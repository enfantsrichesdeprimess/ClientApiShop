# Интернет-магазин на Vue 3

SPA приложение интернет-магазина с авторизацией, каталогом товаров, корзиной и оформлением заказов.

## Стек технологий

- **Vue 3** — основной фреймворк
- **Vuex 4** — управление состоянием
- **Vue Router 4** — навигация
- **Fetch API** — работа с бэкендом
- **CSS3** — стилизация (без дополнительных фреймворков)


## Функциональные требования

### Роли пользователей
- **Гость** (не авторизован) — может просматривать каталог
- **Клиент** (авторизован) — может добавлять товары в корзину, оформлять заказы, просматривать свои заказы

### Экраны приложения

1. **Каталог товаров** (/) — список всех товаров
2. **Регистрация** (/register) — форма создания аккаунта
3. **Вход** (/login) — форма авторизации
4. **Корзина** (/cart) — товары, добавленные в корзину
5. **Оформленные заказы** (/orders) — история заказов пользователя

### Особенности корзины
- Одинаковые товары группируются с указанием количества
- Возможность увеличить/уменьшить количество
- Возможность удалить товар
- Оформление заказа (доступно только если корзина не пуста)

## API

Бэкенд доступен по адресу: `http://lifestealer86.ru/api-shop`

### Основные эндпоинты

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/products` | Получение списка товаров |
| POST | `/login` | Авторизация пользователя |
| POST | `/register` | Регистрация нового пользователя |
| GET | `/cart` | Получение корзины |
| POST | `/cart` | Добавление товара в корзину |
| PUT | `/cart/{id}` | Обновление количества товара |
| DELETE | `/cart/{id}` | Удаление товара из корзины |
| POST | `/order` | Оформление заказа |

### Форматы ответов

**Успешный ответ:**
```
{
  "data": {
    // запрашиваемые данные
  }
}
```
**Ошибка:**
```
{
  "error": {
    "code": 403,
    "message": "Текст ошибки"
  }
}
```

# Установка и запуск
## Требования

- Node.js 16+
- npm 8+

**Установка зависимостей**
```bash
npm install
```
**Запуск в режиме разработки**
```bash
npm run serve
```

**Сборка для прода**
```bash
npm run build
```

Готовые файлы появятся в папке `dist/`
